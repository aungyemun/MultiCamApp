# MultiCamApp Directory Structure

This file summarizes the current project layout and identifies source files, generated release artifacts, local tools, and cleanup-safe areas.

## Root Layout

```text
MultiCamApp\
  MultiCamApp.exe          Dev launcher that starts dist\MultiCamApp.exe
  README.md                Project overview, quick start, citation, and attribution
  INSTALLATION.md          Install, offline test, and developer setup guide
  DIRECTORY_STRUCTURE.md   This file
  LICENSE.md               Full license agreement (9 sections)
  THIRD_PARTY_NOTICES.md   Third-party component notices
  CITATION.cff             Machine-readable citation metadata (CFF v1.2.0)
  SECURITY.md              Security policy and responsible use
  CONTRIBUTING.md          Contribution guidelines
  CHANGELOG.md             Release changelog (user-facing)
  global.json              .NET SDK version pin
  source\                  C# WPF app, launcher, and tests
  dist\                    Generated published app bundle used by Setup.exe
  installer\               Inno Setup script, build script, Setup.exe, VC++ redist, installer.zip
  scripts\                 Build, diagnostic, maintenance, packaging, and setup scripts
  runtime\                 Staged vendor runtime helpers and FFmpeg tools
  tools\                   Local developer toolchain
  docs\                    Architecture, user, release, license, and changelog docs
  data\                    Temporary scratch/download area
```

`.agents` may exist as a local tool folder. It is not part of the release package.

`dist\` is regenerated by `scripts\build\build_release.ps1` (invoked via `installer\build_release.bat`). `installer\MultiCamApp_{version}_Setup.exe` (e.g. `MultiCamApp_2.0.5.338_Setup.exe`) is the end-user offline installer — never a bare `Setup.exe`. `installer\installer.zip` is the shareable package containing the Setup.exe installer and the important user docs.

## Local Dependencies

Developer/build tools:

- `tools\dotnet\` — local .NET SDK
- `tools\python\` — local Python for release and audit scripts
- `tools\inno\` — local Inno Setup compiler
- `tools\nuget-packages\` — local NuGet package cache

Runtime/vendor files:

- `runtime\ffmpeg\ffprobe.exe` — staged into `dist\runtime\ffmpeg\`; used for offline Video Verification
- `runtime\ffmpeg\ffmpeg.exe` — staged into `dist\runtime\ffmpeg\`; shipped to end users, powers the on-demand Deep Verify per-frame MD5 duplicate-frame check. The installer's final validation fails if it's missing.
- `runtime\vc_runtime\vc_redist.x64.exe` — mirrored to `installer\vc_redist.x64.exe`
- `installer\vc_redist.x64.exe` — bundled inside the Setup.exe installer

## Generated And Cleanup-Safe Areas

These folders/files can be regenerated:

- `source\**\bin\`
- `source\**\obj\`
- `scripts\diagnostics\VerifyFolderCli\bin\`
- `scripts\diagnostics\VerifyFolderCli\obj\`
- `.build\`
- `dist\logs\`
- root `logs\`
- `release_audit\`

Do not delete these unless intentionally rebuilding or refreshing release assets:

- `dist\`
- `installer\*_Setup.exe`
- `installer\installer.zip`
- `installer\vc_redist.x64.exe`
- `runtime\ffmpeg\ffprobe.exe`
- `runtime\vc_runtime\vc_redist.x64.exe`
- `tools\`
