# diagnostics/

This folder stores local study outputs generated by diagnostic scripts in the Windows Camera Behavior Study phase.

**This folder should not be committed to version control if it contains machine-specific data.**

---

## .gitignore Recommendations

Add the following lines to your `.gitignore` to prevent machine-specific output files from being committed:

```
diagnostics/*.txt
diagnostics/*.json
diagnostics/*.csv
diagnostics/*.log
```

The `diagnostics/README.md` itself may be committed.

---

## Expected Output Files

Files generated by `scripts/diagnostics/scan_windows_camera_environment.ps1`:

| File | Description |
|---|---|
| `windows_camera_environment_report.txt` | Human-readable environment report (Windows, CPU, RAM, GPU, cameras, USB, power plan, GPU scheduling, camera privacy) |
| `windows_camera_environment_report.json` | Machine-readable JSON version of the same report |
| `dxdiag_multicam.txt` | DirectX diagnostic report from `dxdiag /t` (if dxdiag is available) |

Files planned for future phases (not generated yet):

| File | Description |
|---|---|
| `camera_capabilities.json` | Per-camera capability report from the future Camera Capability Scanner |
| `camera_capabilities.txt` | Human-readable camera capability summary |

---

## What Is NOT Generated in This Phase

- No video validation output
- No ffprobe output
- No recording test output
- No camera FPS test results
- No recorded video files

---

## How to Generate the Environment Report

Run from the project root:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\diagnostics\scan_windows_camera_environment.ps1
```

After running, paste relevant findings into:

- `docs/windows_camera_behavior_study.md` — Section 7 (Local Environment Scan Summary)
